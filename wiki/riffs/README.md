# Riffs

Exploratory thinking. Half-ideas. Things that might be wrong. Each riff carries a status tag:

- **`draft`** — first pass, not yet stress-tested
- **`developing`** — multiple revisions, gaining conviction
- **`tested`** — validated against reality; usually a step away from becoming a spec or decision

Riffs don't commit anyone. They're allowed to be wrong. When one graduates — or gets discarded — the reasoning goes into `wiki/decisions/`.

## Riff block grammar

```markdown
<!-- riff id="unique-id" status="draft|developing|tested" could_become="tweet, blog, talk, ADR, spec" -->
Exploratory thinking here. Speculative, might be wrong.
<!-- /riff -->
```

Don't clean up riff blocks. They're working state, not polish.
